Quick Start

Data Attributes (zero JS)

Add data-hwgl-element to any container. The library auto-initializes on page load.

<div data-hwgl-element
     data-hwgl-source="photo.jpg"
     data-hwgl-shape="circle"
     data-hwgl-color-mode="mono"
     data-hwgl-interaction="sparkle"
     style="width: 100%; height: 400px;">
  <h1>Your content here</h1>
</div>

Video Source

Pass a video URL to data-hwgl-source for looping video halftone effects.

<div data-hwgl-element
     data-hwgl-source="video.mp4"
     data-hwgl-interaction="sparkle"
     style="width: 100%; height: 400px;">
</div>

Webcam Source

Use "webcam" as the source to request camera access on load.

<div data-hwgl-element
     data-hwgl-source="webcam"
     data-hwgl-interaction="warp"
     style="width: 100%; height: 400px;">
</div>

JavaScript API

const ht = new HalftoneWebGL({
  container: '#my-element',
  source:    'photo.jpg',
  shape:     'circle',
  colorMode: 'mono',
  interaction: 'sparkle',
});

// Update at runtime
ht.set('frequency', 60);
ht.set({ shape: 'diamond', colorMode: 'cmyk' });

// Load new source
await ht.loadSource('video.mp4');

// Clean up
ht.destroy();

JavaScript API

Constructor

const ht = new HalftoneWebGL({
  container: '#my-element',  // CSS selector or DOM element (required)
  source: 'photo.jpg',       // any config option can be passed here
  frequency: 50,
  shape: 'diamond',
  interaction: 'sparkle',
});

Methods

.set(key, value) / .set({ ... })

Update one or more configuration options at runtime. Changes apply immediately.

ht.set('frequency', 60);
ht.set({ shape: 'diamond', colorMode: 'cmyk', contrast: 2.5 });

.loadSource(src)

Load a new image, video, or webcam source. Returns a Promise that resolves when the source is ready.

await ht.loadSource('photo.jpg');      // image URL
await ht.loadSource('video.mp4');      // video URL (.mp4, .webm, .ogg)
await ht.loadSource('webcam');         // request webcam access
await ht.loadSource(myVideoElement);   // HTMLVideoElement
await ht.loadSource(myImageElement);   // HTMLImageElement

.resize()

Manually trigger a resize. Called automatically via ResizeObserver — rarely needed.

.pause() / .resume()

Stop or start the animation loop.

ht.pause();   // stop rendering
ht.resume();  // start rendering

.snapshot(type?, quality?)

Capture the current frame as a data URL.

const png = ht.snapshot();                          // PNG (default)
const jpg = ht.snapshot('image/jpeg', 0.92);         // JPEG at 92% quality

// Download as file
const a = document.createElement('a');
a.href = ht.snapshot();
a.download = 'halftone.png';
a.click();

.destroy()

Clean up all resources — stops animation, removes the canvas, releases video/webcam streams, clears event listeners.

ht.destroy();

Events

Subscribe with .on(event, fn), unsubscribe with .off(event, fn).

ht.on('ready', () => {
  console.log('First frame rendered');
});

ht.on('sourceload', () => {
  console.log('Source texture loaded');
});

ht.on('resize', ({ w, h }) => {
  console.log(`Resized to ${w}x${h}`);
});

ht.on('error', (err) => {
  console.error('Error:', err);
});

Color Modes

Four color rendering modes. Set via colorMode option or data-hwgl-color-mode attribute.

mono — single color dots
auto — source image colors
cmyk — 4-channel separation
duotone — two-color blend

Color Mode Details

  • mono — Dots are rendered in a single color on top of bgColor. Classic halftone look.
  • auto — Each dot uses the source image's color at that position. Full-color halftone.
  • cmyk — Four halftone channels (Cyan, Magenta, Yellow, Key/Black) with independent screen angles via cmykAngles. Authentic print halftone.
  • duotone — Shadows use colorB, highlights use color, mid-tones blend between them.

Interactions

Ten mouse-driven interaction effects. Set via interaction option or data-hwgl-interaction attribute. Adjust the area with radius and intensity with strength.

none No mouse interaction. Static halftone rendering.
reveal Fades to the original source image under the cursor.
magnify Enlarges dot size near the cursor.
warp Pushes the dot grid outward from the cursor position.
ripple Animated sine-wave distortion radiating from the cursor.
vortex Rotates the grid around the cursor in a swirl pattern.
colorShift Applies a hue rotation to dots near the cursor.
focus Increases dot frequency (sharpens) near the cursor.
comet Leaves a glowing trail behind the cursor with velocity-based stretching. Uses trailFade.
sparkle Leaves a sparkling trail with per-dot random animation. Uses trailFade.

Content Overlay

Place any HTML inside the halftone container. Children are auto-promoted above the canvas — text is selectable, buttons are clickable.

<div data-hwgl-element
     data-hwgl-source="bg.jpg"
     data-hwgl-interaction="sparkle"
     style="height: 400px;">

  <!-- These sit above the canvas automatically -->
  <h1>Heading</h1>
  <p>Paragraph text — fully selectable.</p>
  <button>Click me</button>

</div>

How it works:

  • The canvas is injected with position: absolute; pointer-events: none
  • Child elements get position: relative; z-index: 1
  • Buttons, links, and [data-hwgl-pause] elements pause the interaction on hover

Custom Pause Elements

Add data-hwgl-pause to any element to pause the interaction when hovered. Or customize the selector:

new HalftoneWebGL({
  container: '#hero',
  pauseSelector: 'a, button, .my-interactive-element',
});
